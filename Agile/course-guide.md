# راهنمای اجرای دوره

## مخاطب و پیش‌نیاز

دوره برای Scrum Master، Product Owner، Project Manager، Jira administrator، Delivery Manager و لیدهای فنی طراحی شده است. دسترسی به یک Jira sandbox مفید است، اما برای مطالعه ضروری نیست. برای بخش‌های Dashboard و Automation به Project admin یا همکاری Jira admin نیاز خواهید داشت.

## الگوی پیشنهادی هر روز

| بخش | زمان | شیوه |
|---|---:|---|
| مطالعهٔ مفهومی | 45 دقیقه | README و فایل‌های اصلی روز |
| تحلیل نمونه | 30 دقیقه | یک گزارش یا Board واقعی |
| Lab عملی | 45 تا 60 دقیقه | ساخت خروجی همان روز |
| بازتاب | 15 دقیقه | ثبت فرض، ریسک و سؤال |

## نحوهٔ استفاده از دوره

1. از روز صفر شروع کنید و یک پروژهٔ آزمایشی را تا انتهای دوره نگه دارید.
2. در هر روز فقط یک یا دو KPI را به‌صورت واقعی پیاده کنید؛ تعداد زیاد KPI ارزش نمی‌سازد.
3. قبل از رفتن به روز بعد، خروجی روز را با چک‌لیست [daily-labs.md](daily-labs.md) تأیید کنید.
4. روز ۸ را با ارائهٔ پروژهٔ نهایی تمام کنید.

## ابزارهای اختیاری

- Jira Software برای Board، Sprint و گزارش‌های پایه
- Confluence یا Markdown برای Definition of Done و تصمیم‌ها
- eazyBI / ابزار تحلیلی مشابه فقط وقتی گزارش داخلی Jira کافی نیست
- GitHub/GitLab و ابزار CI/CD برای DORA metrics

ابزار جایگزین روش فکر کردن نیست؛ ابتدا سؤال تصمیم‌گیری را بنویسید، سپس ابزار را انتخاب کنید.
