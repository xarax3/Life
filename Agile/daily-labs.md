# تمرین‌ها و چک‌لیست روزانه

| روز | Lab | تحویل‌دادنی | شرط عبور |
|---:|---|---|---|
| 0 | Board آزمایشی | Workflow و Definition of Done | یک Issue تا Done حرکت کرده باشد |
| 1 | Reporting | Filter ذخیره‌شده و Dashboard سه‌Gadgetی | هر Gadget یک سؤال دارد |
| 2 | Sprint health | Reliability، Scope Change و Filter کارهای باز | واحد اندازه‌گیری یکسان است |
| 3 | Flow analysis | Control Chart یا تحلیل Cycle Time | یک گلوگاه با شواهد مشخص شده است |
| 4 | Forecast | Capacity sheet و بازهٔ Forecast | فرض‌ها و عدم‌قطعیت نوشته شده‌اند |
| 5 | Kanban/DevOps | WIP policy و یک Flow/DORA dashboard | معیار برای رتبه‌بندی افراد نیست |
| 6 | Scrum admin | Backlog، Board و Sprint policy | نقش‌ها و دسترسی‌ها روشن‌اند |
| 7 | Governance | یک Rule تست‌شده و Data Dictionary | Audit log و Owner دارد |
| 8 | Capstone | Portfolio view و ارائهٔ 15 دقیقه‌ای | Outcome، ریسک و تصمیم بعدی واضح‌اند |

## قالب بازتاب پایان روز

```text
چه چیزی یاد گرفتم؟
کدام داده یا فرض نامطمئن است؟
کدام تصمیم با این خروجی بهتر می‌شود؟
اقدام کوچک بعدی چیست و Owner آن کیست؟
```
