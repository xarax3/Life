# Lab 0 — ساخت Board آزمایشی

## سناریو

برای محصول فرضی «ثبت‌نام آنلاین مشتری»، یک Board بسازید. هدف: کار از ایده تا تحویل قابل مشاهده باشد.

## مراحل

1. یک پروژهٔ آزمایشی Jira بسازید یا از پروژهٔ sandbox استفاده کنید.
2. ستون‌های `Backlog`، `Ready`، `In Progress`، `Review` و `Done` را تعریف کنید.
3. برای `In Progress` یک WIP limit آزمایشی انتخاب کنید.
4. سه Story و یک Bug بسازید؛ برای هر Story Acceptance Criteria بنویسید.
5. Definition of Done را در صفحهٔ پروژه یا Confluence ثبت کنید.
6. یک Issue را از ابتدا تا Done ببرید و زمان‌های انتظار را یادداشت کنید.

## معیار پذیرش

- Board وضعیت هر Issue را بدون سؤال از افراد نشان می‌دهد.
- حداقل یک Policy برای ورود به Done دارید.
- یک Blocker یا وابستگی ساختگی را روی Issue ثبت کرده‌اید.
- نام Statusها با Dashboardهای آینده سازگار است.
