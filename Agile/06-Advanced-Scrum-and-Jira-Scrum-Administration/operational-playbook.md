# 144. Backlog Health Check

قبل از Sprint Planning بررسی:

## Top Items

باید:

* Priority مشخص داشته باشند.
* Description کامل داشته باشند.
* Estimate شده باشند.

---

## Backlog Quality

بررسی:

```
Too Large Items

Missing Acceptance Criteria

Unknown Dependencies

Old Issues
```

---

# 145. Backlog Refinement Checklist

برای هر Story:

```
✓ Business Value

✓ User Need

✓ Acceptance Criteria

✓ Dependencies

✓ Estimate

✓ Technical Understanding

✓ Ready Status
```

---

# 146. Sprint Goal Management

Sprint Goal باید:

## کوتاه باشد

مثال:

```
Improve Customer Payment Experience
```

---

## قابل اندازه‌گیری باشد

مثال:

```
Enable Online Payment For Mobile Users
```

# 147. Sprint Scope Control

# 150. Root Cause Analysis

برای مشکلات تکراری:

روش‌ها:

## 5 Why

مثال:

Problem:

```
Release Delay
```

Why?

```
Testing Late
```

Why?

```
Too Many Features
```

```
Poor Planning
```
